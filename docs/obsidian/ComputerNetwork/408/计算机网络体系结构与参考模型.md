# 计算机网络体系结构与参考模型

## 计算机网络分层结构

![](assets/net-arch/expressage_layer.jpg)

### 分层的基本原则

1. 每层都实现一种相对独立的功能，降低大系统的复杂度

2. 各层之间的接口自然清晰，易于理解，相互交流尽可能少

3. 各层功能的精确定义独立于具体的实现方法，可以采用最合适的技术来实现

4. 保持下层对上层的独立性，上层只能通过接口**单向**使用下层提供的服务

5. 整个分层结构应当能促进标准化工作

!!! info
    - **实体**是指任何可发送或接收信息的硬件或软件进程，通常是特定模块的软件模块。在网络分层结构中，第 $n$ 层的活动元素通常称为**第 $n$ 层实体**。

    - 不同机器上的相同层称为**对等层**，同一层的实体称为**对等实体**。
    
    ![](assets/net-arch/layer_rela.jpg)

    - 第 $n$ 层向第 $n + 1$ 层提供**服务**（包括在其以下的各层提供的服务），第 $n$ 层的实体称为**服务提供者**，第 $n + 1$ 层的实体称为**服务用户**，后者通过**接口**向前者请求服务。

    ![](assets/net-arch/layer_rela_1.jpg)

### 协议

这里的协议指**网络协议**（*Network Protocol*），是控制对等实体之间通信的规则的集合，可简单理解为通信计算机双方事先约定的一种通信规则、通信标准，是**水平的**。

#### 协议三要素

1. **语法**: 数据与控制信息的格式。如协议控制信息（首部）部分占几个字节、每个字节的含义、协议的数据部分最多有多少字节等。

2. **语义**: 需要发出何种控制信息、完成何种动作以及做出何种响应。例如，协议需要明确规定发送方发完数据后，接收方是否需要“应答”、如何“应答”等。

3. **同步**（或**时序**）: 执行各种操作的条件、时序关系等，即事件顺序的详细说明。例如，发送方每发送完一组数据后，等待接收方发出确认，在收到确认后再发送下一组数据。

### 接口

在[分层的基本原则](#分层的基本原则)中已经有所提及，即**同一节点内相邻两层实体交换信息的逻辑接口**，又称为**服务访问点**（*Service Access Point*，SAP）。

### 服务

服务指**下层为紧邻的上层**提供的功能调用，是**垂直的**。
