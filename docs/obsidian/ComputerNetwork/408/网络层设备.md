# 网络层设备

## 冲突域和广播域

这节是“二层/三层设备对比题”的基础。

- **冲突域（Collision Domain）**

    - 在共享介质下，多个节点同时发送会产生冲突的范围

    - 现代交换式以太网中，交换机端口通常把冲突域细分到“每端口”

- **广播域（Broadcast Domain）**

    - 二层广播帧（如 ARP Request）能够到达的范围

    - **路由器会隔离广播域**（跨网段不转发二层广播）

    - VLAN 可以在二层进一步划分广播域（理解即可）

!!! tip
    **交换机解决冲突域细分**；**路由器负责广播域隔离 + 跨网段转发**。

## 路由器的组成和功能

### 路由器的核心功能

路由器做两件事：

- **控制平面**：运行路由协议，生成路由表并下发转发表

- **数据平面**：按转发表高速转发分组（最长前缀匹配）

### 典型转发流水线

以 IPv4 为例：

1. 输入端口收到二层帧，校验并去二层头

2. 读取目的 IP，查 FIB（最长前缀匹配）得到输出端口/下一跳

3. TTL 减 1（若减到 0 丢弃并可能返回 ICMP）

4. 重新封装下一跳二层帧（需要下一跳 MAC，通常用 ARP/ND 得到）

5. 输出端口发送

!!! warning
   路由器转发时**换二层头**是常态，但 IP 层的源/目的地址通常不变（NAT 例外）。

## 路由表与分组转发

### 路由表 vs 转发表

- **路由表（RIB）**：控制平面产生的“路由信息集合”（可能包含多条候选）

- **转发表（FIB）**：真正用于高速转发的表（通常是路由表的精选/优化版本）

### 最长前缀匹配（LPM）

在 CIDR 下，可能同时匹配多个前缀，选择“前缀最长”的那条（更具体的路由优先）。

## 网络层设备对比

| 设备 | 工作层次 | 转发依据 | 能否隔离广播域 |
|:---:|:---:|:---:|:---:|
| 集线器 Hub | 物理层 | 无（比特转发） | 否 |
| 二层交换机 | 数据链路层 | MAC 表 | 否（VLAN 例外，理解即可） |
| 路由器 | 网络层 | IP 前缀（FIB） | **是** |

## 408 高频易错点清单

- 路由器跨网段转发依据是 **IP 前缀**，不是 MAC

- ARP 广播不会跨路由器：路由器隔离广播域

- “换二层头、查三层表”是跨网段转发的标准动作
