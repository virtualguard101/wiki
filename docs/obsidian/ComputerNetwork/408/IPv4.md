## 1. IPv4 数据报（首部）必考字段速记

IPv4 首部最小 20B（无选项）。

- **版本（Version）**：4
- **首部长度（IHL）**：单位 4B，最小 5（即 20B）
- **总长度（Total Length）**：首部 + 数据，单位字节
- **标识 / 标志 / 片偏移（ID / Flags / Fragment Offset）**：分片相关（见 3.3）
- **生存时间（TTL）**：每过一台路由器减 1，为 0 丢弃，抑制环路
- **协议（Protocol）**：上层协议号（如 ICMP=1，TCP=6，UDP=17）
- **首部校验和**：仅覆盖 IPv4 首部；转发时 TTL 变化会导致校验和重算
- **源/目的地址**

## 2. IPv4 编址（分类编址 → 子网 → CIDR）

### 2.1 分类编址（了解为主，但常用于“历史/对比”）

- **A 类**：1.0.0.0–126.0.0.0（首位 0）
- **B 类**：128.0.0.0–191.255.0.0（首位 10）
- **C 类**：192.0.0.0–223.255.255.0（首位 110）
- **D 类**：224.0.0.0–239.255.255.255（组播）
- **E 类**：240.0.0.0–255.255.255.255（保留/实验）

> 408 常考点不是让你死背范围，而是：**分类编址浪费地址 → 引出 CIDR/聚合**。

### 2.2 子网划分（Subnetting）

- **目的**：把一个网络号再细分，便于组织与路由管理
- **核心**：**子网掩码**（或前缀长度）把地址划分为：网络前缀 + 主机号
- **计算**：
  - **可用主机数**：\(2^{h}-2\)（全 0 网络地址、全 1 广播地址不可用）
  - **子网数**：由借用的主机位数决定

### 2.3 无分类编址 CIDR（408 高频）

- 表示法：`a.b.c.d/x`（x 为前缀长度）
- **最长前缀匹配（LPM）**：转发查表选择匹配位数最多的项
- **路由聚合（超网）**：把多个连续网络前缀合并成更短的前缀，减小路由表规模

## 3. 关键机制：ARP / ICMP / 分片

### 3.1 ARP（地址解析协议）

- **目标**：已知同一链路上的 IPv4 地址，求对应 MAC 地址
- **工作方式**：广播请求 + 单播应答；主机缓存 ARP 表（有老化）
- **易错点**：ARP 工作在“链路层语义上”，但服务于网络层的地址到二层地址映射

### 3.2 ICMP（差错报告与询问）

- **作用**：网络层控制消息（错误报告 + 诊断）
- **典型应用**：
  - `ping`：ICMP Echo Request/Reply
  - `traceroute`：利用 TTL 递减触发 ICMP Time Exceeded（不同实现对探测报文类型略有差异）
- **常见差错**：目的不可达、超时（TTL=0）、参数问题、重定向（了解）

### 3.3 IPv4 分片与重组（高频计算/概念题）

- **触发**：某段链路 MTU < IP 数据报长度
- **关键字段**：
  - **ID**：同一原始数据报的所有分片相同
  - **MF（More Fragments）**：除最后一片外均为 1
  - **片偏移**：单位 **8B**
- **重组位置**：**目的主机**（不是中间路由器）

## 4. NAT（网络地址转换）（408 高频理解点）

- **动机**：IPv4 地址不足，内网私有地址复用公网地址
- **常见私有地址段**：10/8，172.16/12，192.168/16
- **NAPT（端口复用）**：多台主机共享一个公网 IP，通过端口区分连接
- **影响**：
  - 破坏端到端可达性（影响某些 P2P/入站连接）
  - 需要配合端口映射/打洞等技术（了解）

---

## 5. 408 高频易错点

- **TTL 的意义**：不是“存活时间（秒）”，是**每过一跳减 1**的跳数限制语义
- **片偏移单位**：**8 字节**（很容易算错）
- **ICMP 不是 TCP/UDP 上层**：它本身就是网络层控制协议（封装在 IP 里）
