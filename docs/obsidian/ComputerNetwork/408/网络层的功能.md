## 1. 这一层“解决什么问题”（一句话版）

网络层的目标是：**在“多个异构网络互连”的环境里，把分组从源主机跨网送到目的主机**；核心手段是 **IP 编址 + 路由选择 + 分组转发**。

---

## 2. 网络层提供的服务模型（408 常考）

- **无连接的数据报服务（Internet / IP）**
  - **特点**：每个分组独立选路；尽力而为（best effort），不保证可靠、按序、时延上界
  - **优点**：网络内部简单、健壮、扩展性好
  - **影响**：可靠性/按序等通常由传输层（TCP）或应用层实现

- **面向连接的虚电路服务（了解，对比题常用）**
  - **特点**：先建立连接（确定路径/状态），再按连接转发
  - **对比点**：是否需要“连接建立与维护状态”、是否更易提供 QoS

> 记忆抓手：**IP = 数据报（无连接）**；“虚电路”更多用于对比理解。

## 3. 核心功能拆解（按“考点颗粒度”）

### 3.1 编址与分片相关（与 IPv4/IPv6 笔记联动）

- **逻辑地址（IP 地址）**：标识“网络 + 主机（或接口）”
- **子网划分 / CIDR**：用掩码/前缀把地址空间结构化，服务于路由聚合与转发表规模控制
- **分片与重组（IPv4）**
  - **动机**：跨链路 MTU 不同
  - **要点**：IPv4 可在路由器分片；重组在目的主机
  - **IPv6 对比**：IPv6 路由器不分片（通常源端用 PMTUD + 分片扩展首部）

### 3.2 路由选择（控制平面）与分组转发（数据平面）

- **路由选择（routing）**
  - **做什么**：运行路由算法/路由协议，计算“到各目的网络的下一跳”
  - **结果**：形成路由表/转发表（FIB）
  - **范围**：
    - **域内路由（IGP）**：如 RIP、OSPF（见“路由算法与路由协议.md”）
    - **域间路由（EGP）**：BGP

- **分组转发（forwarding）**
  - **做什么**：根据目的地址查表，决定输出端口/下一跳，完成实际转发
  - **典型流程**：
    - 查转发表（最长前缀匹配）
    - 处理 TTL/Hop Limit、校验等
    - 选择下一跳并封装到下一段链路层帧

> 高频区分：**路由选择 = 算路（慢、协议驱动）**；**转发 = 走路（快、查表转发）**。

### 3.3 异构网络互连

- **异构**：不同链路层技术（以太网/Wi‑Fi/PPP…）、不同 MTU、不同带宽与时延
- **互连关键**：路由器作为三层设备，屏蔽底层差异，让“IP 视角”看起来像一个逻辑互联网络

### 3.4 拥塞控制（网络层视角：理解 + 可能考名词）

- **拥塞**：网络中分组过多导致排队时延激增、丢包上升、吞吐下降
- **两类思想**：
  - **开环控制**：设计时预防（如合理的路由、资源预留思想）
  - **闭环控制**：运行时检测并反馈（如通过丢包/ECN 等信号让端系统调节）

> 408 更常把“拥塞控制”作为体系结构理解点，具体机制更多在传输层 TCP（但你要知道网络层也有相关信号/策略）。

---

## 4. 408 高频易错点（背诵清单）

- **“网络层负责可靠传输”**：错。IP 尽力而为；可靠性主要在 TCP/链路层局部重传等
- **“路由器负责重组分片”**：错。IPv4 分片可在路由器做，但 **重组在目的主机**
- **“路由选择=转发”**：错。算路 vs 查表转发
