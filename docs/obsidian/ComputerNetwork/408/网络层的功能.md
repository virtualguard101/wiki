# 网络层的功能

## 异构网络互联

异构网络互联（Internetworking）指：用网络层把不同链路层技术的网络（以太网/Wi‑Fi/PPP/…）连接成一个逻辑互联网络。

异构体现在：

- 物理/链路层技术不同（帧格式、MAC、接入方式不同）

- MTU 不同（导致 IPv4 可能分片）

- 带宽/时延/丢包特性不同

网络层通过统一的 **IP 编址与转发** 把差异“屏蔽”起来：上层只关心“IP 能到达”。

## 路由与转发

这一节 408 必考“区分题”。

- **路由选择（Routing，控制平面）**

    - 做什么：运行路由算法/协议（RIP/OSPF/BGP 等），计算到各目的网络的下一跳

    - 产物：路由表（RIB）/转发表（FIB）

    - 特点：相对慢、协议驱动、在拓扑变化时收敛

- **分组转发（Forwarding，数据平面）**

    - 做什么：对到来的每个分组做**最长前缀匹配**查表，决定输出端口/下一跳并转发

    - 特点：相对快、实现常硬件化（ASIC/TCAM）

!!! tip
    **routing = 算路；forwarding = 查表走路。**

## 网络层提供的两种服务

### 虚电路

虚电路（Virtual Circuit，VC）是“面向连接”的网络层服务模型（更多用于对比理解）：

- **建立连接**：先建立一条逻辑连接/路径（可在沿途保存状态）

- **沿连接转发**：后续分组按连接标识转发

- **优点**：更容易做资源预留/QoS（理解）

- **代价**：网络内部复杂，需要维护连接状态；某节点故障可能影响连接恢复

### 数据报

数据报（Datagram）是 Internet 的主线：IP 提供 **无连接、尽力而为**的数据报服务。

- 每个分组独立选路/转发（无连接）

- 不保证：可靠、按序、时延上界

- “可靠/按序”等通常由 **TCP** 或应用层机制完成

## SDN的基本概念

SDN（Software-Defined Networking）在“网络层功能”中常以体系结构对比出现：

- **传统网络**：控制平面与数据平面紧耦合（路由器自己跑协议、自己转发）

- **SDN**：把控制平面逻辑集中/可编程化（控制器），数据平面设备按流表转发

!!! tip "408需要掌握的程度"
    - 能说清“控制平面 vs 数据平面”的分离思想

    - 理解它与“路由选择/转发”这对概念的对应关系

## 拥塞控制

### 拥塞控制的基本概念

拥塞：网络中分组过多 → 排队时延增大、丢包上升、吞吐下降。

与“流量控制”区分（常见混淆点）：

- **流量控制**：端到端（发送方别把接收方压垮）

- **拥塞控制**：面向网络（别把网络压垮）

### 拥塞控制的方法

按经典划分（408 常用口径）：

- **开环控制**（预防型）

    - 在设计阶段用规则避免拥塞：合理路由、资源规划、准入控制思想（理解即可）

- **闭环控制**（反馈型）

    - 运行中检测拥塞并反馈：通过丢包、显式拥塞通知（ECN）等信号让端系统/网络调整

!!! tip 
    做题时通常不要求写出复杂算法，而是是否能把“拥塞是什么、和流控区别是什么、开环/闭环是什么”说清楚。

## 408高频易错点清单

!!! tip "408高频易错点清单"
    - 网络层（IP）是尽力而为：**不保证可靠交付**

    - 路由选择 ≠ 分组转发：控制平面 vs 数据平面

    - 拥塞控制 ≠ 流量控制：网络层面 vs 端到端接收能力
