# 移动IP

## 移动IP的概念

移动 IP（Mobile IP）要解决的核心矛盾：

- IP 地址在路由意义上与“所在网络前缀”绑定

- 主机跨网移动后，如果 IP 变了，上层连接（尤其 TCP）通常会断

因此移动 IP 的目标是：**主机移动时仍能用“归属地址”对外通信**，并把报文转发到“当前位置”。

### 关键术语

- **MN（Mobile Node，移动节点）**：会移动的主机

- **HA（Home Agent，归属代理）**：在归属网，为 MN 做“转交/转发”

- **FA（Foreign Agent，外部代理）**：在外部网提供转交服务（部分体系中可省略，理解即可）

- **Home Address（归属地址）**：MN 在归属网的固定地址（对外标识）

- **CoA（Care-of Address，转交地址）**：MN 在外部网的临时地址（标识当前位置）

!!! tip
   归属地址像“身份证号”，CoA 像“当前住址”，HA 像“转信的邮局”。

## 移动IP的通信过程

把通信过程记成“三步主线”：**发现 → 注册 → 隧道转发**。

### 1）发现（Agent Discovery / CoA 获取）

- MN 进入外部网后，获得 CoA

    - 常见方式：通过外部代理通告/配置获得（做题时知道“能得到 CoA”即可）

### 2）注册（Registration）

- MN 将 CoA 注册到 HA

- 让 HA 维护“归属地址 → CoA”的绑定关系

### 3）隧道转发（Tunneling / Encapsulation）

!!! tip "“报文怎么走”"
    1. 通信对端仍把分组发往 MN 的 **归属地址**

    2. 分组到达归属网后被 **HA 截获**

    3. HA 对原 IP 数据报进行**再封装**（隧道），发往 CoA

    4. 在外部网由 FA（或 MN 自身）**解封装**，交付给 MN

---

## 三角路由

现象：路径可能变成

对端 → HA（归属网）→ CoA（外部网）→ MN

代价：

- 路径变长、时延上升

- HA 成为性能瓶颈与单点压力

（“路由优化”了解即可，408 更常问三角路由的原因与影响）

---

## 408 高频易错点清单

- 对外标识仍是 **归属地址**，不是 CoA

- 核心机制是 **隧道再封装**：把“发往归属地址”的包转交到 CoA

- 三角路由不是“算路算法差”，而是“需要经过 HA 转交”的体系结构副作用
