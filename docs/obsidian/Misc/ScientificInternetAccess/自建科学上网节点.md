# 自建科学上网节点

受网络环境影响[^1]，近期常用[透明代理](透明代理.md)节点频繁断线，平时用的五个梯子三个日常断线，还有一个稳定性差，遂考虑自建科学上网节点。

## 法律问题[^2]

从严格的法律角度上来说，翻墙访问被禁止的网页或者服务涉嫌违法。违法行为如何处罚，要根据具体的情节和危害程度来定。一般而言，对于绝大多数翻墙用户来说，如果只是浏览信息，并没有散布行为，一般不予处罚。如果对于禁止的网页和服务，进行散布和传播，理论上构成犯罪。

如果提供翻墙服务，则一定是违法且犯罪，并一定会承担法律后果。

翻墙看看国外的信息，低调点，问题不大，但是请不要触碰危害到国家利益的底线，拒绝讨论和散布有关邪教和敏感的政治话题。

教授他人翻墙的行为涉嫌犯罪。

| 协议 | 原理 | 特点 | 缺点与风险 |
|:------:|:------:|:------:|:------:|
| **Shadowsocks** | 客户端将流量加密后发往境外服务端，服务端解密再以 SOCKS 语义向目标转发；现代实现多用 AEAD 类算法保证完整性与机密性。 | 部署与配置相对简单，资源占用小，客户端生态成熟；本质是「加密代理通道」，不包含复杂路由引擎。 | 裸跑时链路上往往不像海量 HTTPS，可区分性相对更高；依赖密码/算法版本，老旧配置脆弱；服务端 IP 可被重点封锁；不同实现可能存在统计或时序层面的指纹差异。 |
| **VMess** | V2Ray / Xray 等框架中的入站协议之一，以 UUID 等完成身份校验，载荷加密，并可与多种传输层、TLS、路由规则组合。 | 路由分流、多出站链、WebSocket / gRPC 等与 CDN 或 TLS 伪装组合时灵活度高，适合复杂拓扑。 | 配置维度高，一环错误（证书、SNI、路径等）易削弱伪装效果；无外层 TLS 时自定义握手与帧特征更易进入针对性分析；协议与实现相对「重」，连接形态更丰富，可能更易被统计模型关注。 |
| **Trojan** | 对外呈现为标准 TLS（常为 TLS over TCP）；通过认证后的连接承载代理流量，未通过认证时可回落为正常 HTTPS 站点内容。 | 设计上与「访问一个 HTTPS 网站」同构，易用 443 端口与合法证书对齐大众流量形态；对端口扫描、简单主动探测可呈现「像真站」的失败面。 | 仍面临 TLS 指纹、SNI、证书链与站点行为是否一致等问题；出口 IP 被关照时协议本身无法规避；配置或回落页不当可能留下可识别特征。 |
| **Hysteria 2** | 基于 QUIC（通常走 UDP），在 QUIC 之上承载协议；结合为高丢包、高延迟链路优化的拥塞控制（如 Brutal），强调恶劣网络下的吞吐。 | 流量族谱上接近现实网络中大量存在的 QUIC / HTTP/3；在部分网络环境下 UDP 443 与 TCP 策略不同步时可能带来体验差异；烂链路上有效带宽往往优于传统 TCP 代理。 | 运营商对 UDP 或 QUIC 限速、阻断时直接失效；「像 QUIC」不等于不可识别，针对性策略会收窄优势；仍需证书、域名与客户端栈等配套，与其他 TLS 类方案类似存在指纹与一致性要求。 |

实际可达性还与出口 IP、本地运营商策略、配置是否规范等因素强相关。
