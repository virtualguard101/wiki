# Go命名规范

>[Go语言命名规范 | Go语言进阶之路](https://golangstar.cn/go_series/go_base/go_naming_standards.html)

在Golang中，变量、常量、全局函数、结构体、接口、方法等要素都需要遵循一定的命名规范，以提高代码的可读性和可维护性。

## 封装命名

对于需要暴露给外部使用的标识符，应使用驼峰命名法，**首字母大写**；而不需要暴露给外部使用的标识符，**首字母小写**。

这里的暴露与不暴露类似面向对象编程中的`public`与`private`。前者可以被包外的元素访问，后者则只能在包内访问。

## 包名称

全部为小写字母，无下划线，尽量简短。尽可能不与标准库重名，且最好与目录名保持一致。

## 文件名

简短有意义；必须为小写字母；允许使用下划线，但尽可能避免，且头尾不能有下划线。

最后一点是因为go编译器会将头尾有下划线的文件识别成具有特定功能的文件，如`_test.go`文件会被识别为测试文件，不会编译到最终的可执行文件中；另外一些系统文件也是由下划线开头的。
